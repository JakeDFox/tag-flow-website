Getting started

Install TagFlow from the Shopify App Store. Once installed you'll be taken directly into the app. From there, click + New rule to create your first tagging rule. No configuration is required before creating rules.
No. TagFlow works on every Shopify plan — Basic, Grow, Advanced, and Plus. This is one of the main reasons TagFlow exists: Shopify Flow (which offers similar functionality) is only available on Shopify Plus. TagFlow brings automated order tagging to everyone.
TagFlow uses Shopify webhooks, which means tagging happens in real time — typically within a few seconds of an order being placed. You can confirm this by checking the audit log after placing a test order.
From the Rules page, click + New rule. Give your rule a name, enter the tag you want applied (e.g. vip-order), then add one or more conditions. All conditions must match for the tag to be applied. Click Save rule and it becomes active immediately.

Rules & conditions

TagFlow currently supports the following condition fields:
  • cart_total — the order subtotal in your store currency
  • shipping_country — the 2-letter country code of the shipping address
  • customer_email — the customer's email address
  • payment_gateway — the payment method used (e.g. shopify_payments, paypal)
  • line_item_product_tags — tags on any product in the order
  • customer_tag — tags on the customer placing the order
  • item_count — total number of line items in the order
  • discount_code — discount code applied at checkout
Yes. You can add as many conditions as you need to a single rule. All conditions must match for the tag to be applied — this is an AND relationship. For example, you can create a rule that only fires when cart total is over $200 AND the shipping country is not US.
Rules are evaluated top to bottom in the order shown on the Rules page. Multiple rules can match and apply tags to the same order — TagFlow does not stop after the first match. If you want a rule to take priority, drag it to the top of the list.
Yes. Use the toggle next to each rule on the Rules page to enable or disable it. Disabled rules are skipped entirely and will not appear in the audit log.
Shopify tags are case-sensitive and can contain letters, numbers, hyphens, and underscores. We recommend lowercase with hyphens for readability — for example vip-order, intl-ship, or route-wh-b. Avoid spaces in tag names.

Audit log

The audit log records every rule evaluation for every order shows you the last 100 entries. Each entry shows the order number, the rule that ran, the tag that was applied (or would have been applied), the source (webhook or manual sync), the status (applied, skipped, or failed), the reason if skipped, and the timestamp.
Skipped means the rule ran but did not apply a tag. This is normal and expected behaviour. Common skip reasons include:
  • Conditions not met — the order didn't match the rule's conditions
  • Tag already applied — the tag was already on the order, so no action was needed
Skipped entries are not failures — they're confirmation that the rule ran and made a decision.
Audit log entries are retained for 90 days. If you need to export your log for record-keeping, use the Export CSV button on the audit log page.
Success rate is calculated as applied ÷ (applied + failed). Skipped entries are excluded from the calculation — a skipped rule is working correctly, not failing. A 100% success rate means every rule that matched an order successfully applied its tag.

Manual sync

Manual sync lets you run your active rules against existing orders. This is useful when you install TagFlow on a store that already has orders, or when you create a new rule and want to apply it to recent orders retroactively. Every tag applied during a sync is recorded in the audit log with the source marked as manual.
No. Before applying a tag, TagFlow checks whether it already exists on the order. If the tag is already there, the rule is logged as skipped with the reason "tag already applied" and no duplicate is created. It is safe to run manual sync multiple times.
You can sync orders up to 60 days back. For stores with a large order volume, syncing a smaller date range is faster. You can run multiple syncs across different date ranges if needed.

Billing

TagFlow is $14/month after a free 7-day trial. No credit card is required to start the trial. Billing is handled through Shopify — charges appear on your regular Shopify invoice.
No. TagFlow does not limit orders per month or charge based on volume. One flat price covers everything regardless of how many orders your store processes.
Uninstalling TagFlow immediately stops all tagging. Your subscription is cancelled automatically through Shopify and you will not be charged again. Tags that have already been applied to orders remain on those orders — uninstalling does not remove previously applied tags.
If you have an issue with TagFlow that we can't resolve, email us at hello@tagflow.run and we'll sort it out. Shopify processes all billing, but we're happy to work with you directly on any billing concerns.

Troubleshooting

First check the audit log — search for the order number to see exactly what happened. Common causes:
  • The rule is disabled — check the toggle on the Rules page
  • Conditions aren't matching — check the skip reason in the audit log
  • The condition value doesn't match exactly — country codes are 2 letters uppercase (e.g. US, CA), payment gateways are lowercase (e.g. shopify_payments)
If the order doesn't appear in the audit log at all, email us and we'll investigate.
Place a test order using the payment method you want to condition on, then check the audit log. The audit log shows the exact values TagFlow sees for each order. Common gateway values include shopify_payments, paypal, shop_pay, and bogus_gateway for test orders.
Shopify can take a few seconds to reflect new tags in the admin UI. Try refreshing the order page. If tags still aren't showing after a minute, email us at hello@tagflow.run with the order number and we'll look into it.
Each rule creates its own audit log entry per order — so if you have 5 active rules, each order will generate 5 entries (some applied, some skipped). This is expected. If you see the same rule applied twice to the same order, email us as that may indicate a webhook retry issue.
